# Gamestation_SC95F8736

基于 **SC95F8736 / 赛元 51 系列 8 位单片机** 的迷你游戏机嵌入式工程，适配 **Keil C** 开发环境。项目实现了单人飞机射击、双人联机对战、时间显示、温湿度显示、闹钟提醒和背景音乐等功能，同时包含 TFT 屏幕驱动、SPI / UART / I2C 通信、按键扫描、图像绘制、游戏逻辑与外设驱动等嵌入式开发内容。

本项目可作为单片机游戏机设计、嵌入式图形界面开发、外设通信调试和小型交互式嵌入式系统的参考工程。

---

## 项目简介

本项目围绕 SC95F8736 单片机进行嵌入式综合应用开发，设计并实现了一款具有多种功能的迷你游戏机系统。系统通过 TFT 彩屏进行图形显示，使用按键和摇杆完成人机交互，并结合多种片上外设与外部模块完成游戏、时间、温湿度和闹钟功能。

主要功能包括：

- 单人飞机射击游戏；
- 双人联机飞机对战；
- 敌机、子弹、BOSS 等像素图形绘制；
- 碰撞检测、得分统计、生命值管理和关卡机制；
- 主菜单与多模式 UI 页面；
- 时间显示、温湿度显示与闹钟提醒；
- ESP8266 网络时间获取；
- SHT30 温湿度传感器数据读取；
- PWM 无源蜂鸣器背景音乐；
- TFT 屏幕 SPI 驱动；
- UART 双机通信与 I2C / TWI 外设通信。

项目采用模块化代码结构，包含芯片配置、底层驱动、固件库和用户应用逻辑，便于后续维护、功能扩展和移植到同系列单片机平台。

## 功能展示

### 1. 单人飞机射击游戏

单人模式下，玩家控制我方战机移动并发射子弹，系统随机生成敌机与 BOSS。游戏通过判断飞机和子弹之间的相对位置完成碰撞检测，根据击中敌机或被敌方子弹击中更新分数和生命值。

主要机制：

- 我方战机移动控制；
- 子弹生成、刷新与边界清除；
- 敌机随机生成与位置更新；
- BOSS 关卡设计；
- 碰撞检测；
- 得分与生命值显示；
- 游戏结束页面与最终分数显示。

### 2. 双人联机对战

双人模式下，两块单片机通过 UART 串口连接。双方玩家实时发送按键和摇杆信息，另一方接收后在本地 TFT 屏幕中更新敌方飞机位置，实现双人飞机对战。

主要机制：

- UART0 双机通信；
- 按键与摇杆信息编码传输；
- 串口接收中断处理；
- 双方飞机位置同步；
- 镜像显示逻辑；
- 双人模式下的碰撞、得分与生命值判断。

### 3. 时间、温湿度与闹钟

系统支持时间、温湿度和闹钟功能。时间部分通过 ESP8266 连接网络接口获取时间信息，并结合 TIMER4 定时器进行时间递增。温湿度部分通过 SHT30 模块读取环境数据，并在 TFT 屏幕上实时显示。闹钟功能基于 TIMER4 定时器触发，当达到设定时间时，屏幕弹出 `TIME OUT!!!` 提示窗口，数秒后自动返回原界面。

主要功能：

- ESP8266 STA 模式配置；
- 网络时间获取；
- TIMER4 定时计时；
- 闹钟时间设置；
- 到时弹窗提醒；
- SHT30 温湿度读取；
- TFT 实时显示时间和环境数据。

### 4. 背景音乐

系统使用 PWM 控制无源蜂鸣器发声，通过定时器控制节拍、PWM 输出频率控制音调，实现游戏背景音乐效果。

## 软件设计

### 1. TFT 屏幕驱动

项目通过硬件 SPI 协议驱动 TFT 屏幕，并封装了基础图形绘制函数，包括：

- 画点；
- 画线；
- 字符显示；
- 数字显示；
- 区域刷新；
- 游戏图像绘制。

为了提升显示效率，项目采用“只刷新变化区域”的动态显示方案，避免每帧重绘整屏造成闪烁和性能浪费。

### 2. 图像绘制

游戏中的战机、敌机、子弹和 BOSS 均使用像素点和线段绘制。通过计算各图形相对于中心点的坐标，在 TFT 屏幕上生成游戏角色图像。

典型图形包括：

- 28 × 28 像素我方战机；
- 16 × 16 像素敌方飞机；
- 2 × 2 像素子弹；
- 44 × 40 像素 BOSS。

### 3. UI 页面

系统包含多个 UI 页面：

- 主菜单页面；
- 时间与温湿度显示页面；
- 单人游戏开始页面；
- 双人联机游戏页面；
- 游戏运行页面；
- 游戏结束页面；
- 闹钟提示弹窗。

主菜单通过 `SELECT` 按键切换模式，通过 `START` 按键进入或退出当前模式。

### 4. 游戏逻辑

单人游戏核心逻辑包括：

- 敌机生成；
- 玩家移动；
- 子弹发射；
- 子弹位置刷新；
- 敌机位置刷新；
- 碰撞检测；
- 得分统计；
- 生命值扣除；
- 游戏结束判断；
- 关卡与 BOSS 机制。

双人游戏核心逻辑包括：

- UART 数据发送；
- UART 接收中断；
- 对方按键与摇杆信息解析；
- 敌方飞机位置镜像刷新；
- 双人对战中的碰撞和得分判断。

### 5. 时钟、温湿度与闹钟

- ESP8266 通过 AT 指令配置为 STA 模式；
- 通过 TCP 客户端方式访问网络时间接口；
- TIMER4 定时器用于时间递增和闹钟触发；
- SHT30 通过 I2C / TWI 协议读取温湿度数据；
- 到达闹钟时间后自动弹出 `TIME OUT!!!` 提醒。

### 6. 背景音乐

项目通过 PWM 控制无源蜂鸣器，通过修改 PWM 输出频率控制音调，并由定时器中断控制节拍，实现动态背景音乐播放。

## 快速开始

### 1. 克隆仓库

```bash
git clone git@github.com:WU-ss/Gamestation_SC95F8736.git
```

如果你的 GitHub 仓库名为 `Gamestation`，可以使用：

```bash
git clone git@github.com:WU-ss/Gamestation.git
```

### 2. 打开 Keil 工程

打开 Keil C，选择：

```text
Project → Open Project
```

然后进入：

```text
Keil_C/Project/
```

选择对应的 Keil 工程文件，例如：

```text
.uvproj
.uvprojx
```

### 3. 编译工程

在 Keil 中点击：

```text
Build
```

工程已集成基本固件库、芯片配置和底层驱动代码，正常情况下无需额外配置即可编译。

### 4. 获取固件文件

编译完成后，在以下目录查看输出文件：

```text
Keil_C/Output/
```

常见输出包括：

```text
.hex
.bin
```

### 5. 烧录运行

使用适配 SC95F8736 的烧录工具，将生成的固件文件烧录到单片机中。上电后，系统进入游戏机程序，可通过按键切换游戏、时钟、温湿度与闹钟等功能。

## 注意事项

1. 烧录前请确认芯片型号、供电电压、晶振配置和引脚连接是否与工程配置一致。
2. 若编译失败，请检查 Keil 版本、工程路径、芯片支持包和头文件路径配置。
3. 若 TFT 屏幕无显示，请检查 SPI 接线、屏幕初始化参数、供电和背光控制。
4. 若按键或摇杆控制异常，请检查 IO 口配置、按键扫描逻辑和硬件焊接。
5. 若双人联机异常，请检查 UART 波特率、TX/RX 交叉连接、接收中断和数据编码格式。
6. 若 ESP8266 获取时间失败，请检查 WiFi 配置、AT 指令流程、网络接口和串口通信。
7. 若温湿度数据显示异常，请检查 SHT30 地址、I2C / TWI 时序和模块供电。
8. 若蜂鸣器无声，请检查 PWM 输出引脚、定时器配置和蜂鸣器连接。
